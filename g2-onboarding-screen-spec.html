G2-1 / Onboarding Screen Spec

今作ろうとしているオンボーディング画面仕様

これは最終デザインではなく、G3でUI画像を作る前の「画面仕様の見える版」です。ここで見るべきなのは、外見生成のワクワク感、質問の軽さ、サンプル経路とカスタム経路の分岐、初回フィードバック後の課金導線です。

何を見せているか

オンボード全体のワイヤー。レビューを書き込みながら判断する仕様確認用。

勝ち筋

食事管理設定より先に「自分だけの食事パートナーが生まれる」体験を出す。

軽い経路

作るのが面倒な人は、顔アップの6サンプルから選んですぐ進める。

重い経路

こだわる人は、参考URL、顔、髪、服、関係性をタップ中心で作れる。

用語検討

現時点の推奨は、アプリ内の総称を「食事パートナー」に寄せることです。「トレーナー」はフィットネス寄りで、恋人・友達・先輩のような関係性と少し衝突します。一方で「栄養士」は資格・監修の印象が強いため、MVPでは避けます。

食事パートナー

食事管理と伴走の両方が伝わる。UIの正式総称として最有力。

相棒

感情訴求が強い。短い見出しやCTAでは使いやすいが、食事アプリ文脈は補足が必要。

食事コーチ

行動改善は伝わるが少し硬い。数字管理やPFC文脈では相性がよい。

トレーナー

既存方針と近いが、運動アプリに寄って見える。関係性候補として残す。

食事ナビ

軽いがキャラクター性は弱い。機能名向き。

栄養士

信頼感はあるが資格・医療監修の印象が強い。現段階のUI総称にはしない。

外部リサーチ要点

AIパートナー系は「外見・関係性・話し方」を先に作り、食事管理系は「写真/記録・目標・体重・フィードバック」を短く聞きます。このMVPでは、見た目生成の待ち時間に食事入力を進め、両方の強みをつなぎます。

AIパートナー系

Replika、Nomi、Kindroid、Candy AI系は、相手の見た目・関係性・性格を先に選ばせ、会話開始前に「自分の相手」という感覚を作る。

画像/キャラ生成系

絵柄、年齢、髪、顔、服、参考イメージを選択式で扱うパターンが多い。自由記述は補助に回す方が軽い。

食事管理系

あすけん、カロミル、MacroFactor系は、食事記録、体重/目標、栄養フィードバックが中心。初回価値は「入力直後の返答」で出す。

参考: Replika / Nomi / Kindroid / Candy AI / あすけん / カロミル

フロー全体

サンプル経路は短く、カスタム経路は顔と人物像の作成に寄せます。どちらも最後は初回食事報告、フィードバック、継続したくなる行動選択まで進めます。参考URLがある場合は早い段階で読み込み、以降の選択肢を仮選択済みにします。

001 分岐自分で作る / サンプルから選ぶ
002 サンプル顔アップ6人から選ぶ
003-007 外見絵柄・年齢・国籍・参考URL・顔・髪・服
007T 生成開始裏側で見た目生成を開始
008A 目的維持 / 減量 / 増量 / 食生活改善
008B 関係性距離感と呼ばれ方
009 生成約60秒、食事入力を同時進行
012-014 価値直前の食事・返信・継続導線

開始

食事パートナーを作るか、サンプルから選ぶ。

カスタム経路

絵柄 → 見た目の性別/年齢/国籍/参考URL → 顔 → 髪 → 服/アクセサリー/NG → 生成開始 → 食事目的/体重 → 関係性/呼ばれ方 → 食事入力 → ご対面。

サンプル経路

6人から選ぶ → 食事目的/体重 → 関係性/呼ばれ方 → 初回食事入力。外見調整、任意補足、参考URLは出さない。

共通ゴール

初回食事入力 → 生成/選択した相手とご対面 → 必要なら1回だけ調整 → 初回食事フィードバック → 応援画像 → 継続したい行動を選ぶ。

レビュー記入欄

各画面の右側に個別レビュー欄があります。下の横断レビュー欄には、画面単位では判断しづらい論点を書いてください。記入内容はこのブラウザ内に自動保存されます。

SCR-ONB-004
見た目の性別・年齢・国籍
カスタム

            

              
仕様
  • 「性別の印象」ではなく「見た目の性別」として聞く。

  • 参考URLが任意だと分かるか。
  • 読み込み中でも操作できることが伝わるか。

カスタム

生成中
カスタム
